# v0.5.0 - Agent Task Persistence with Beads

## Problem

Sage currently stops after each model response and waits for user input. When given a complex task that requires multiple steps, it:
1. Makes one request
2. Executes one tool call
3. Gets a follow-up response
4. Stops and waits

This means sage "forgets" to keep working on multi-step tasks.

## Solution

Add an **agent loop** that uses beads as persistent task memory:
1. Model identifies tasks needed
2. Tasks are stored in beads
3. Agent loop processes tasks until done
4. Progress visible to user throughout

## Architecture

### New Module: `src/sage/agent.scm`

```scheme
(define-module (sage agent)
  #:use-module (sage config)
  #:use-module (sage logging)
  #:use-module (sage ollama)
  #:use-module (sage session)
  #:use-module (sage tools)
  #:export (*agent-tasks*
            agent-start
            agent-loop
            agent-pause
            agent-status
            task-create
            task-complete
            task-list
            task-current
            with-task-context))
```

### Task Lifecycle

```
User Request
    |
    v
[Model analyzes request]
    |
    v
[Creates tasks in beads]  <-- bd create --prefix sage-
    |
    v
[Agent loop begins]
    |
    +---> [Pick next task]
    |         |
    |         v
    |     [Send task to model]
    |         |
    |         v
    |     [Execute tool calls]
    |         |
    |         v
    |     [Mark task complete] --> bd close
    |         |
    |         v
    |     [More tasks?] --yes--> loop
    |         |
    |        no
    |         |
    v         v
[Report completion to user]
```

### Beads Integration

Tasks are stored as beads issues with `sage-` prefix:
```bash
# Create task
bd create "Implement foo function" -t task --prefix sage- -d "..."

# List pending tasks
bd list --prefix sage- --status open

# Complete task
bd close sage-abc --reason "Done: implemented foo()"
```

### Agent Modes

1. **Interactive** (default): Agent pauses after each task for user review
2. **Autonomous**: Agent continues until all tasks complete or error
3. **Yolo**: Autonomous + no permission prompts (dangerous tools auto-approved)

### Session Integration

Agent state stored in session:
```scheme
(define *agent-state*
  `(("mode" . "interactive")      ; interactive | autonomous | yolo
    ("current-task" . #f)         ; beads issue ID
    ("pending-tasks" . ())        ; list of beads issue IDs
    ("completed-tasks" . ())
    ("max-iterations" . 10)       ; safety limit per request
    ("iteration" . 0)))
```

### New Tools

**sage_task_create**: Create a task for the agent
```scheme
(register-tool
 "sage_task_create"
 "Create a task for the sage agent to work on"
 '(("type" . "object")
   ("properties" . (("title" . ...)
                    ("description" . ...)
                    ("depends_on" . ...))))
 agent-task-create)
```

**sage_task_complete**: Mark current task complete
```scheme
(register-tool
 "sage_task_complete"
 "Mark the current task as complete"
 '(("type" . "object")
   ("properties" . (("result" . ...)
                    ("notes" . ...))))
 agent-task-complete)
```

**sage_task_list**: List pending tasks
```scheme
(register-tool
 "sage_task_list"
 "List all pending sage tasks"
 '(("type" . "object")
   ("properties" . ()))
 agent-task-list)
```

### REPL Changes

Add commands:
- `/agent [mode]` - Show/set agent mode
- `/tasks` - List current tasks
- `/pause` - Pause agent loop
- `/continue` - Continue agent loop

Modify `handle-chat` to:
1. After model response, check for task creation
2. If tasks created and mode != interactive, enter agent loop
3. In agent loop, keep calling model with "continue working on tasks" prompt
4. Exit loop when: all tasks done, error, pause requested, or max iterations

### Status Display

During agent loop, show progress:
```
[sage] Working... (task 2/5: "Add error handling")
  |
  +-- Tool: write_file -> OK
  +-- Tool: read_file -> OK
  |
[sage] Task complete. Next: "Write tests"
```

1. Create `src/sage/agent.scm` with task state management
2. Add `sage_task_*` tools to `tools.scm`
3. Add beads CLI integration (shell out to `bd`)
4. Add `/agent`, `/tasks`, `/pause`, `/continue` commands to REPL

6. Add status display during agent loop
7. Bump version to 0.5.0
8. Test with multi-step task: "Create a new tool that generates UUIDs"

## Safety Limits

- **max_iterations**: 10 per user request (configurable via SAGE_MAX_ITERATIONS)
- **max_task_depth**: 3 levels of subtask nesting
- **auto_pause_on_error**: Stop loop if tool returns error
- **dangerous_tool_pause**: Always pause before dangerous tools in autonomous mode
