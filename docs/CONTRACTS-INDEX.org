Single entry point to the documents that govern guile-sage behaviour. Three
kinds of doc:

- *Living guides* — describe the current implementation; must be kept in sync.
- *Contracts* — behavioural boundaries with (where they exist) an invariant
  test that fails if the contract is broken.
- *Specs / schemas* — formats and forward-looking designs (some not yet
  implemented; flagged as such).

Point-in-time *reports* live under =docs/reports/= (index:
[[file:reports/README.org][reports/README.org]]) and are never edited after
creation. Diagrams follow
[[https://wal.sh/research/diagram-style-guide#org010b7ab][wal.sh/research/diagram-style-guide]].

*How to scan this tree:* every =docs/*.org= declares its state on a top-of-file
=#+STATUS:= line — =Current= (matches the implementation), =Design= (forward-looking,
not yet built), =Frozen= (a snapshot, not maintained), or =Disputed= (a known
spec/code mismatch). Run =grep '^#+STATUS' docs/*.org= to see the whole tree's
state at a glance. What was *tested against a running system*, and when, is the
live-runs ledger in [[file:VALIDATION-INDEX.org::#live-runs][VALIDATION-INDEX.org]].

* Engineering boundaries (hard constraints)

The non-negotiables, with the full rationale + reproducible experiments in
[[file:CHAOS-TESTING.org][CHAOS-TESTING.org]]:

1. *Build/test:* =gmake= only (BSD =make= no-ops the GNU Makefile); runtime
   =guile3=/=guild3=; gates =gmake check / lint-code / lint-org / lint-contracts
   / lint-docs= (the last fails on hardcoded RFC1918 IPs in living docs).
2. *Subprocess safety:* never =system*= / =open-pipe*= / =open-input-pipe=
   (they SIGSEGV in long-lived multi-threaded Guile); libc =(system …)= only.
3. *HTTP resilience:* curl =--connect-timeout 5=; probe fails fast; chat retries
   429/5xx with backoff (=SAGE_HTTP_MAX_RETRIES=).
4. *Provider errors:* normalize to =[<provider> <label>: <msg>]=, never throw a
   raw body.
5. *Taint:* all external bytes are untrusted; route via =execute-tool= →
   =guard-tool-result= → =wrap-tool-result=.
6. *.env precedence:* =config-load-dotenv= overrides shell env — edit =.env=.

* Navigating

- Implementing a feature → [[file:ARCHITECTURE.org][ARCHITECTURE.org]] + the relevant contract.
- Deploying / configuring → [[file:DEPLOYMENT.org][DEPLOYMENT.org]] + [[file:../.env.template][.env.template]].
- Adding a tool → [[file:TOOLS-GUIDE.org][TOOLS-GUIDE.org]] + register in =src/sage/tools.scm=.
- Touching transport / errors / spawn → re-read the boundaries above and [[file:CHAOS-TESTING.org][CHAOS-TESTING.org]].
- Why a past decision was made → =docs/reports/= and =git notes=.
