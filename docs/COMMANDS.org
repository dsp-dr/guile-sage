* Design Principle

Every feature accessible three ways:
1. *CLI flag* - =sage --flag= (for scripts, one-shot)
2. *Slash command* - =/command= (interactive REPL)
3. *Scheme API* - =(sage-function)= (programmatic)

* Custom Slash Commands (v0.3)

Like Gemini CLI, support custom commands in:
- =~/.config/sage/commands/*.scm= (user scope)
- =.sage/commands/*.scm= (project scope)

#+begin_src scheme
;; Example: .sage/commands/test.scm
(define-command test
  #:description "Run project tests"
  #:args '((pattern "Test pattern" #:optional))
  (lambda (args)
    (execute-tool "run_tests" args)))
#+end_src

Becomes =/test pattern= in REPL.

* @ File References (v0.3)

Shorthand for adding files to context:

| Syntax               | Meaning                     |
|----------------------+-----------------------------|
| =@./src/foo.scm=     | Add single file             |
| =@./src/=            | Add directory (recursive)   |
| =@./src/**/*.scm=    | Glob pattern                |
| =@https://url=       | Fetch and add URL           |

* Command Implementation Pattern

#+begin_src scheme
;;; Standard pattern for dual CLI/REPL commands

;; 1. Core function (Scheme API)
(define* (feature-function #:key (option default))
  "Documentation."
  ...)

;; 2. Slash command wrapper
(define (cmd-feature args)
  "Slash command /feature."
  (feature-function #:option (parse-args args))
  #t)

;; 3. Register command
(define *commands*
  `(("/feature" . ,cmd-feature)
    ...))

;; 4. CLI flag handling in main.scm
(define (process-args args)
  (cond
   ((member "--feature" args)
    (feature-function #:option (get-flag-value args "--feature")))
   ...))
#+end_src
