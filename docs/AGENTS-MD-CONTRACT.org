* Purpose and scope

This document is a contract specification for how =guile-sage= discovers,
parses, and applies =AGENTS.md= files from project directories. =AGENTS.md=
is the sage equivalent of Claude Code's =CLAUDE.md= — a project-local
configuration file that shapes agent behavior, tool permissions, and
operating conventions without modifying source code.

It captures:

- Discovery order across five candidate locations
- Merge semantics when multiple =AGENTS.md= files are found
- Structured section parsing (headings that map to runtime behavior)
- Tool permission extraction and enforcement
- Three-way access: CLI flag, slash command, Scheme API
- Numbered invariants to be enforced in tests

The existing implementation lives in =src/sage/config.scm:256-282=
(=find-agents-md= and =load-agents-md=). The context module at
=src/sage/context.scm:66-73= loads =SAGE.md= but never calls
=load-agents-md=, representing a gap this contract addresses.

Related beads issues: =guile-sage-grv= (Agent Modes), =guile-sage-w9p=
(CLI/REPL command synergy), =guile-sage-51k= (Custom slash commands).

Key relationships:

- =AGENTS.md= files are authored by the developer and read by sage.
- Discovery is file-system based — no network, no database.
- The parsed configuration affects which tools are exposed to the LLM
  and which operating mode sage uses.

* 3. Contracts and invariants

The implementation and its tests MUST enforce each invariant below.
Each has a short code (A##) so tests can reference it.

** Discovery

| ID  | Description                                                    | Test strategy          |
|-----+----------------------------------------------------------------+------------------------|
| A01 | =find-agents-md= checks exactly 5 locations in this order:    | Unit: mock filesystem  |
|     | workspace, XDG project, XDG config, XDG data, ~/.sage/        |                        |
| A02 | Only files that pass =file-exists?= are returned              | Unit: temp dir fixture |
| A03 | If no AGENTS.md exists anywhere, =find-agents-md= returns =()= | Unit: empty temp dir  |

** Merge semantics

| ID  | Description                                                    | Test strategy          |
|-----+----------------------------------------------------------------+------------------------|
| A04 | Multiple files are concatenated in discovery order             | Unit: 2-file fixture   |
| A05 | Each file's content is prefixed with =# From: <path>=         | Unit: check prefix     |

** Structured section parsing

| ID  | Description                                                    | Test strategy          |
|-----+----------------------------------------------------------------+------------------------|
| A06 | =## Tool Permissions= heading maps to safe/unsafe tool lists  | Unit: parse mock file  |
| A07 | =## Mode= heading maps to operating mode (yolo, plan, normal) | Unit: parse mock file  |
| A08 | =## Conventions= heading is passed through as system context   | Unit: parse mock file  |

** Application

| ID  | Description                                                    | Test strategy          |
|-----+----------------------------------------------------------------+------------------------|
| A09 | Tool permission restrictions from AGENTS.md are applied        | Integration: verify    |
|     | BEFORE the first chat turn                                     | tool list after load   |
| A10 | Malformed AGENTS.md degrades gracefully: log WARN, skip the    | Unit: garbage input    |
|     | malformed section, continue with remaining sections            |                        |
| A11 | =/agents reload= replaces prior AGENTS.md config atomically   | Unit: load, reload,    |
|     | (no partial state between old and new config)                  | verify no interleave   |
| A12 | Nested AGENTS.md in subdirectories are NOT traversed;          | Unit: place file in    |
|     | only the 5 explicit search locations are checked               | subdir, verify missed  |

* 4. Observed divergences

N/A: speculative feature. The structured section parsing (A06-A09) and
the =/agents= slash command (A11) are not yet implemented. The current
code in =config.scm:256-282= handles only discovery (A01-A03) and
concatenation (A04-A05).

The context module (=context.scm:66-73=) loads =SAGE.md= via
=load-agents-context= but never calls =load-agents-md= from
=config.scm=. Wiring this is a prerequisite for any AGENTS.md
functionality to take effect at runtime.

* 5. Implementation notes

** Existing code

- =src/sage/config.scm:256-268= — =find-agents-md=: builds a list of 5
  candidate paths and filters by =file-exists?=. Uses =getcwd= for
  workspace (not the =--workspace= CLI flag).
- =src/sage/config.scm:272-282= — =load-agents-md=: maps over found
  files, reads each with =get-string-all=, prefixes with
  =# From: <path>=, joins with =\n\n=. Returns =#f= if none found.
- =src/sage/context.scm:66-73= — =load-agents-context=: loads =SAGE.md=
  from the workspace directory. This is the function called during REPL
  startup (=repl.scm:371= and =repl.scm:1037=). It does NOT call
  =load-agents-md=.
- =src/sage/config.scm:28-41= — exports =find-agents-md= and
  =load-agents-md= but no consumer currently calls them at startup.

** Gaps to close

1. Wire =load-agents-md= into the REPL startup path (after
   =load-agents-context=).
2. Parse structured sections from the merged content string.
3. Extract tool permissions and apply them to =tools.scm= safe/unsafe
   lists before the first chat turn.
4. Add =--agents-md PATH= CLI flag to =main.scm= option-spec.
5. Add =/agents= slash command family to =repl.scm=.

* 6. Three-way access

| Access method       | Interface                    | Status       |
|---------------------+------------------------------+--------------|
| CLI flag            | =--agents-md PATH=           | Not yet impl |
| Slash command       | =/agents=, =/agents reload=  | Not yet impl |
| Scheme API          | =(agents-md-load)=,          | Partial:     |
|                     | =(find-agents-md)=,          | find/load in |
|                     | =(load-agents-md)=           | config.scm   |

The CLI flag should override discovery and load only the specified file.
The slash command should support:

- =/agents= — show current AGENTS.md sources and parsed config
- =/agents reload= — re-read and re-apply all AGENTS.md files
- =/agents show= — display the merged content

* 7. Test strategy

** Unit tests (=tests/test-agents-md.scm=)

Stub test file with SRFI-64 harness. Each invariant (A01-A12) has a
named =test-assert= placeholder. Tests will be filled in when the
structured parsing is implemented.

** Integration tests

Once the =/agents= command and structured parsing are implemented:

- Verify that a workspace =AGENTS.md= with =## Tool Permissions= section
  restricts the tool list visible to the LLM.
- Verify that =/agents reload= picks up changes to an =AGENTS.md= file
  modified after startup.
- Verify that =--agents-md /path/to/custom.md= overrides the discovery
  order.

** Property-based tests

- Fuzz the section parser with random markdown to confirm A10 (graceful
  degradation).
- Verify that discovery order is deterministic across repeated calls.
