* Objective

Push the cloud Ollama implementation to its limits with real-world complex tasks:
- Deep codebase understanding
- Multi-turn conversations that exhaust context
- Heavy tool usage (read, search, explore)
- Trigger compaction and recovery
- Validate retry logic under load

* Test Philosophy

#+begin_quote
"Grind until we compact, then grind some more."
#+end_quote

Each scenario should:
1. Start with a complex, open-ended question
2. Require extensive tool use (10+ calls)
3. Build context over multiple turns
4. Hit token limits naturally
5. Continue after compaction to verify state preservation

* Test Execution Protocol

** Phase 1: Baseline (No Retry)
1. Run scenarios against cloud
2. Document all failures
3. Capture error types and frequencies
4. Establish baseline metrics

** Phase 2: With Retry Logic
1. Implement retry with exponential backoff
2. Re-run same scenarios
3. Compare failure rates
4. Validate Retry-After header handling

** Phase 3: With Auto-Compaction
1. Enable auto-compaction at 80%
2. Run long scenarios
3. Verify context preservation
4. Measure information retention

** Phase 4: Full Stack
1. Retry + Streaming + Auto-Compaction
2. All scenarios
3. End-to-end validation
4. Performance benchmarks

* Test Harness Structure

#+begin_src scheme
;; tests/stress/scenario-runner.scm

(define-scenario "solidus-state-machine"
  #:description "Extract TLA+ spec from Solidus order lifecycle"
  #:setup (lambda ()
            (ensure-repo "solidus/solidus"))
  #:prompts '("Clone and analyze Solidus order state machine"
              "Find all order states and transitions"
              "What guards protect each transition?"
              "Write a TLA+ specification for this"
              "What edge cases could cause invalid states?")
  #:expected-tools '(git_clone read_file search_files list_files)
  #:min-tool-calls 15
  #:expect-compaction #t)

(define (run-scenario scenario)
  (let ((session (session-create #:name (scenario-name scenario))))
    (for-each (lambda (prompt)
                (format #t "~%>>> ~a~%" prompt)
                (handle-chat prompt)
                (display (format-session-status))
                (when (> (token-usage-percent) 80)
                  (format #t "~%[COMPACTION TRIGGERED]~%")
                  (session-compact!)))
              (scenario-prompts scenario))
    (validate-scenario scenario session)))
#+end_src

* Failure Injection

To validate retry logic, simulate failures:

#+begin_src scheme
;; Wrap http-post with failure injection
(define *failure-rate* 0.1)  ; 10% failure rate

(define (http-post-with-failures url body . args)
  (if (< (random:uniform) *failure-rate*)
      (cons 429 "Rate limited")
      (apply http-post url body args)))
#+end_src
