* Executive Summary

guile-sage is an AI-powered REPL built in Guile Scheme with tool calling capabilities. It provides a Claude Code-like experience for local and cloud LLM providers.

* Key Questions & Answers

** 1. Security Model

*** Q: How is the sandbox implemented?
- *Path Safety*: All file operations check against workspace boundaries
- *Sensitive Files*: Blocks .env, .git/, .ssh/, .gnupg/ access
- *Command Injection*: Shell metacharacters filtered in tool parameters
- *Permission System*: Safe vs Unsafe tools with YOLO mode override
- *34 adversarial security tests* pass against common attack vectors

*** Q: What are the attack surfaces?
| Surface | Mitigation |
|---------+------------|
| Path traversal | Canonicalization + prefix check |
| Command injection | Parameterized commands, shell escaping |
| Sensitive file exposure | Regex blocklist for .env, credentials |
| Temp file races | PID-namespaced temp files, immediate cleanup |
| Eval injection | YOLO mode required, not default |

*** Q: What's NOT protected?
- No OS-level sandboxing (no containers, no seccomp)
- TOCTOU races possible (time-of-check to time-of-use)
- YOLO mode intentionally bypasses all restrictions

** 2. System Prompt / Context

*** Q: How is context managed?
- Sessions stored per-project in XDG directories
- Claude-style path encoding: ~/project → -home-user-project
- AGENTS.md support for project-specific instructions
- 5 compaction strategies with quality evaluation

*** Q: What are the token limits?
| Provider | Default Limit |
|----------+---------------|
| Local (host.lan) | 8,000 tokens |
| Cloud | 4,000 tokens |

** 4. Cost & Infrastructure

*** Q: What's the cost model?
- *Local*: Free (runs on local Ollama)
- *Cloud*: Pay per token to cloud provider
- No SaaS dependencies
- No telemetry or data collection

*** Q: What infrastructure is required?
| Component | Requirement |
|-----------+-------------|
| Runtime | Guile 3.0+ |
| LLM Backend | Ollama (local or cloud) |
| Dependencies | curl (HTTP), standard POSIX |
| Storage | XDG directories (~/.local/share/sage) |
