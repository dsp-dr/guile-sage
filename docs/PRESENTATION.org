* Introduction

** What is guile-sage?

- AI-powered REPL in Guile3 Scheme
- Conversational interface to LLMs
- Tool calling for code manipulation
- Self-modifying capabilities

** Why This Project?

Current multi-agent frameworks (MetaGPT, CAMEL, AutoGen) focus on *orchestration* but lack:

- *Attribution*: Who authorized this agent? Who is accountable?
- *Governance*: How do we audit and trust autonomous agents?
- *Self-hosting*: Can you understand and modify your own agent?

guile-sage explores:
- Self-modifying agents that can improve themselves
- External audit layers (watchdogs outside the main loop)
- Homoiconic language for true meta-programming

#+begin_quote
"The agent that can modify itself is the agent you can trust—because you can see what it's doing."
#+end_quote

/Reference: [[https://wal.sh/research/agentic-2026/][Agentic Systems Research Q1 2026]]/

** Why Guile Scheme?

- *Homoiconicity*: Code is data, data is code (meta-programming)
- *Macros*: Compile-time code generation
- *REPL-native*: Live system modification
- *GNU ecosystem*: Emacs, Guix, standards-compliant

** Differentiation from Python Frameworks

| Feature | MetaGPT/CAMEL/AutoGen | guile-sage |
|---------+-----------------------+------------|
| Self-modification | Not emphasized | Core feature |
| Audit layer | Logging only | External watchdog (planned) |
| Runtime modification | Reload modules | Hot-reload + eval |
| Meta-programming | Limited | First-class (homoiconic) |
| Governance | Cost tracking | Dual watchdog architecture |

* Features

** Tool Categories

*** Safe Tools (Always Allowed)
- read_file, list_files
- git_status, git_diff, git_log
- search_files, glob_files

*** Unsafe Tools (YOLO Mode)
- write_file, edit_file
- git_commit, eval_scheme
- create_tool, reload_module

** Self-Modification

#+begin_src scheme
;; Create new tools dynamically
(execute-tool "create_tool"
  '(("name" . "greet")
    ("description" . "Say hello")
    ("code" . "(lambda (args) \"Hello!\")")))

;; Evaluate arbitrary code
(execute-tool "eval_scheme"
  '(("code" . "(+ 1 2 3)")))
#+end_src

* Safety Model

** Path Safety

#+begin_src scheme
(define (safe-path? path)
  (and (not (string-contains path ".."))
       (not (regexp-exec
             (make-regexp "(\\.env|\\.git/|\\.ssh)")
             path))))
#+end_src

** Permission System

#+begin_src scheme
(define (check-permission tool args)
  (or (member tool *safe-tools*)
      (config-get "YOLO_MODE")
      #f))  ; Default deny
#+end_src
