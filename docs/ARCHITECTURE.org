* Design Philosophy: Tools Over Memory

Sage is NOT tool-agnostic. It explicitly uses file-based tools rather than
in-memory state.

** Why Tools, Not Memory

| In-Memory (BAD)              | File-Based Tools (GOOD)           |
|------------------------------|-----------------------------------|
| TODO list in session state   | bd create/list/close              |
| Notes stored in variables    | git notes, write_file             |
| Context in conversation only | AGENTS.md loaded at startup       |
| Lost on restart              | Persisted in filesystem           |

** Core Tool Categories

Tools are classified into a *safe* tier (always permitted) and an
*unsafe* tier (require =SAGE_YOLO_MODE=). The permission contract is
defined in [[file:adr/0003-security-model.md][ADR-0003]] and enforced by =check-permission= in
=src/sage/tools.scm=. The annotations below mirror that contract.

*** File Operations (Pure Guile)
- =read_file= - Read file content [safe]
- =write_file= - Write/create files [unsafe — needs YOLO]
- =edit_file= - Modify existing files [unsafe — needs YOLO]
- =list_files= - Directory listing [safe]
- =glob_files= - Pattern matching [safe]
- =search_files= - Content search [safe]

*** Git Operations (Shell, TODO: libgit2)
- =git_status= - Working tree state [safe]
- =git_diff= - Show changes [safe]
- =git_log= - Commit history [safe]
- =git_commit= - Create commits [unsafe — needs YOLO]
- =git_push= - Push to remote [unsafe — needs YOLO]
- =git_add_note= - Attach notes to commits [unsafe — needs YOLO]

*** Self-Inspection (Pure Guile)
- =read_logs= - Read sage's own logs
- =search_logs= - Search log patterns

*** Agent Tasks (Beads via shell, TODO: native)
- =sage_task_create= - Create task in beads
- =sage_task_complete= - Mark task done
- =sage_task_list= - List pending tasks
- =sage_task_status= - Agent loop status

** The AGENTS.md Contract

On startup, sage reads =AGENTS.md= and injects it as a system message.
This file tells the model:

1. How to use sage's task system
2. Project-specific workflows
3. Beads issue tracking conventions
4. Landing the plane (commit + push) protocol

The model should follow AGENTS.md instructions, using tools to:
- Track work in beads (not memory)
- Persist changes to files (not conversation)
- Commit and push (not just "ready when you are")

* Shell Code Smell Tracking

Goal: Replace all shell calls with pure Guile or native bindings.

| Module     | Shell Usage              | Status      | Alternative          |
|------------|--------------------------|-------------|----------------------|
| util.scm   | curl for HTTPS POST      | TODO        | (web client) + gnutls|
| tools.scm  | git commands             | TODO        | libgit2 bindings     |
| tools.scm  | grep, find               | TODO        | pure Guile search    |
| agent.scm  | bd commands              | TODO        | beads Guile API      |
| context.scm| (none - pure Guile)      | DONE        | -                    |

* Session State

Sessions are saved as JSON files in =.sage/sessions/=:

#+BEGIN_SRC json
{
  "name": "session-1234567890",
  "model": "qwen3-coder:latest",
  "created": "1234567890",
  "updated": "1234567899",
  "messages": [...],
  "stats": {
    "total_tokens": 5000,
    "input_tokens": 2000,
    "output_tokens": 3000,
    "request_count": 10,
    "tool_calls": 5
  }
}
#+END_SRC

This is file-based persistence, not in-memory.
