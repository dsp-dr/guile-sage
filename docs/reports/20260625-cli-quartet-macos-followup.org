* 2. What the filled columns change

1. *MCP-server mode is still uncontested.* Of four, only *Codex* runs as an MCP
   server. OpenCode ships an *ACP* server (=opencode acp=) and an HTTP =serve= +
   =web= — adjacent, but not MCP. The substrate play (*SAGE-t2l*) loses no ground;
   if anything OpenCode's ACP shows the field reaching for "expose the agent as a
   callable service," which is exactly sage's bet, just via MCP instead of ACP.
2. *Session fork is now unanimous (all =Y=).* Every one of the four branches a
   session — Claude =--fork-session=, Codex =fork=, OpenCode =--fork=, and (now
   binary-confirmed) Droid =--fork <sessionId>=. guile-sage persists + compacts
   but cannot *branch*. This is the one table-stakes verb we're visibly missing.
   (Droid also surprised on multiagent: =--mission= drives separate
   =--worker-model= / =--validator-model= agents — first-class, ahead of sage's
   =P= aq/coordinator; default model is =claude-opus-4-8=.)
3. *Time-travel has two distinct levels.* Everyone resumes/forks *conversations*;
   only Claude Code and (via snapshot dirs) OpenCode/Codex checkpoint *files*.
   sage's compaction is conversation-level only. File-level checkpoint is a real,
   separable capability — and it composes with our sandbox/taint boundary.
4. *OTLP is a sage differentiator we under-sell.* Claude Code emits OTLP; OpenCode,
   Codex, Droid emit *none*. guile-sage already has opt-in OTLP + a JSONL usage
   ledger. Combined with #1 this is the strongest strategic thread below.
5. *Local-first is native here, bolted-on there.* All four can reach Ollama, but
   sage treats Ollama/Gemini/LiteLLM as first-class providers; the others need a
   custom-provider stanza (and Droid's =exec --model= still rejects custom IDs,
   Factory #169/#787/#1061). On a LAN/self-hosted box this is a posture advantage.

* 3. Roadmap — sequenced

Priorities reuse the L3 assets we own; each maps to a bead candidate. (=SAGE-t2l=
and =SAGE-4fl= already exist; the rest are proposed IDs for the maintainer.)

| # | Item | Why now (from the quartet) | Effort | Bead |
|---+------+----------------------------+--------+------|
| P1 | *MCP server mode* — stdio =initialize= / =tools/list= / =tools/call= over =*tools*= | only Codex does it; 2 live consumers on mini (Claude Code, Emacs gptel/mcp.el) | M | =SAGE-t2l= |
| P2 | *Session fork* — branch a persisted session | now unanimous across all 4; sage is the visible gap | S | new =SAGE-fork= |
| P3 | *Telemetry substrate* — when agents call sage-as-MCP-server, sage emits the OTLP they lack | only Claude+sage emit OTLP; satisfies mini's hard nexus requirement *for* the blind agents | S–M | new =SAGE-otlp-sub= |
| P4 | *Sandbox tiers* — adopt Codex =sandbox_permissions= disk-levels + Droid =--auto low/med/high= autonomy gates into RFC-003 / HC-01 | two independent designs converge on tiered confinement | M | =SAGE-4fl=→RFC-003 |
| P5 | *=sage review=* — non-interactive review verb | =codex review= is a discrete table-stakes verb; pairs with our eval lineage | S | new =SAGE-review= |
| P6 | *File-level checkpoint* — snapshot/restore touched files across a turn | Claude file-history + OpenCode/Codex snapshot dirs; composes with taint boundary | M | new =SAGE-ckpt= |
| P7 | *Inspector surface* — =sage serve= read-only web/JSON view of session+ledger | field is adding web (OpenCode =web=, Droid bridge, Claude remote-control); ride P1's loop, don't build a TUI | L | watch |

** Phasing
- *Now (substrate):* P1 then P3. P1 makes sage callable; P3 makes it the *governed,
  observable* substrate — the combination is the moat, not P1 alone.
- *Next (parity, cheap):* P2 + P5 — small, visible, close the "missing verbs" gap.
- *Then (depth):* P4 + P6 — the sandbox/checkpoint capabilities that justify
  "governed substrate" over "yet another agent."
- *Watch:* P7 — only after a third consumer asks; keep the LAN/self-hosted posture.

* 4. Strategic line (one sentence)

The quartet confirms the field is racing on *agent UX* (web surfaces, cloud,
remote control) and leaving *governance + observability* on the floor — so
guile-sage should not chase their UX; it should *be the MCP server they call*
(P1) and *emit the telemetry they don't* (P3), turning its existing L3 assets —
tool registry, sandbox, taint boundary, OTLP ledger — into the substrate other
agents run on.
