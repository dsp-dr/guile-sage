* Executive Summary

*guile-sage* is an AI-powered REPL (Read-Eval-Print Loop) written in Guile3 Scheme
that enables conversational interaction with Large Language Models through tool calling.
Built on the Ollama backend, it provides a self-modifying coding assistant that can
read, write, and execute code within a controlled workspace.

* Key Features

| Feature | Description |
|---------+-------------|
| Tool Calling | 15 built-in tools for file operations, git, search |
| YOLO Mode | Optional unrestricted tool access for power users |
| Session Management | Persistent conversations with context compaction |
| XDG Compliance | Standard directory structure following XDG spec |
| Project Isolation | Per-project sessions (Claude Code-style path encoding) |
| Self-Modification | Tools to create tools, eval code, reload modules |

* Safety Model

** Safe Tools (Always Allowed)
- read_file, list_files
- git_status, git_diff, git_log
- search_files, glob_files

** Unsafe Tools (Require YOLO Mode)
- write_file, edit_file
- git_commit, git_add_note
- eval_scheme, create_tool, reload_module
- run_tests

** Path Safety
- Workspace isolation (no access outside project)
- Parent traversal blocked (..)
- Sensitive files protected (.env, .git/, .ssh/)

* Deployment

** Requirements
- Guile 3.0+
- curl (HTTP client)
- Ollama (local or cloud)

** Quick Start
#+begin_src bash
git clone https://github.com/dsp-dr/guile-sage
cd guile-sage
cp .env.template .env
# Edit .env with your Ollama host
make run
#+end_src

** YOLO Mode
#+begin_src bash
guile3 -L src -c '(use-modules (sage repl)) (repl-start)' --yolo
#+end_src
