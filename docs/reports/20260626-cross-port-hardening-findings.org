Five teams independently implemented the same boundary spec (=spec.org=): the
Guile reference + Python/Go/Janet/Common-Lisp ports (the ports are ephemeral and
private — their VALUE is what they reveal about the spec and the reference impl).
Their independent choices were diffed against guile-sage's own source. Result:
*4 real gaps + 1 partial in guile-sage*, and *1 spec ambiguity* that sent the
ports astray. All actioned below.

** 1 — taint envelope scope (the ambiguity that misled 4 ports)
All four ports routed *MCP-server* =tools/call= results through the taint
envelope (=<tool-result safe="false">= CDATA). guile-sage does NOT
(=mcp-server.scm= returns plain MCP =content[].text=) — and guile-sage is
*right*: that XML envelope is sage's INTERNAL convention for feeding its OWN LLM
(the REPL path); forcing it onto the MCP wire would make a peer client (Claude
Code) render literal =<tool-result>= tags. The spec said results "pass through
the taint choke point" without saying *which* path — so the ports over-applied
it. *Fix = spec*: the taint envelope is for the inbound / own-LLM consumption
path only; as an MCP *server*, sage returns plain protocol content and the
client owns its own trust boundary. (A server-side guard may still sanitize, but
must not impose the envelope.)

** 3 — HTTP 408 (=util.scm=)
=http-retryable-code?= retried 429/5xx only; 408 (Request Timeout) is transient.
Ports retried it. *Fixed*: 408 added to the retryable set.

** 4 — provider error body (=openai/gemini/ollama.scm=)
A non-200 body was embedded verbatim into the =[Provider …]= line — a multi-line
or multi-kilobyte body could inject newlines into the transcript / flood it. The
ports bounded + single-lined it. *Fixed*: new =clean-error-message= (util)
collapses whitespace and caps at 200 chars; applied at all three provider sites.

** 5 — =safe-path?= NUL (=tools.scm=)
NUL-byte rejection was implicit (relied on =canonicalize-path=). Ports rejected
NUL up front. *Fixed*: explicit =(string-index path #\nul)= guard alongside the
=..= check.

** 6 — MCP =-32700= (=mcp-server.scm=)
Malformed JSON was caught and logged to stderr only — silently dropped from the
client's view. Ports replied =-32700 Parse error= (id null). *Fixed*: parse is
now separated from dispatch; a parse failure emits the canonical
=-32700= line.

** 2 — CDATA escape (confirmed-OK)
=escape-cdata-sequence= / =fetch-cdata-safe= already use =]]]]><![CDATA[>= and
are injection-tested. Added a byte-faithful round-trip regression (the ports all
had one; guile-sage's was implicit).

* Round 2 — sage-lua (Lua 5.4), the first port of the *hardened* spec
41/41 property checks pass; it validated round-1's clarifications (408,
single-line bound, NUL, -32700, served-results-plain) AND surfaced more:

- *Oracle leak (security, FIXED in guile-sage):* =tools/call= on an unknown tool
  returned =-32601 "Unknown tool"=, but on an unexposed unsafe tool =-32600
  "Tool not exposed (set SAGE_MCP_EXPOSE_UNSAFE=1)"= — revealing that a gated
  tool *exists* and how to escalate. Now both return the *identical* =-32601=;
  the env-var hint is stderr-only. Regression in =scripts/mcp-smoke.sh=.
- *Spec ambiguities clarified (§2/§3/§4):* =<label>= is provider-normalized
  (NOT byte-identical across providers — the classification is the contract);
  =<msg>= is explicitly a sanitized bounded *excerpt* (not "no body at all");
  classification ≠ retry policy (code 0 is transient-classified but fails fast);
  dotfile blocklist is *path-segment* match (=my.env= ok), =..= is substring;
  bare/relative paths resolve workspace-relative, empty rejected; blank/
  whitespace-only stdin lines are framing (skipped, NOT =-32700=).
- *Porting note (not a spec issue):* Lua =require()= returns a 2nd value
  (loader path) in 5.3/5.4 — silently expands a table; any Lua port will hit it.

* Round 3 — v2 cohort (C++, Ruby, Haskell, Clojure, Scala, Kotlin)
Six ports against the twice-hardened spec; cross-port *agreement* is the signal.
Pass counts: C++ 55 (~45k trials), Ruby 64, Haskell 50 (~64k), Clojure 33,
Scala 31, Kotlin 75. All green, all local.

Fixed in guile-sage (code):
- *=safe-path?= per-token + =/tmp= block (security).* The old substring regex
  over-blocked (=my.env=, =.gitignore=) AND under-protected (=/tmp/.ssh/id_rsa=
  was readable). Pure segment-match (what the ports did) then MISSED the
  secret-bearing =.env.local= — caught by the existing test suite. Final rule is
  *per token*: =.env= or =.env.*=; exact =.git=/=.ssh=/=.gnupg=; applied to ALL
  paths incl =/tmp=. (Neither substring nor exact-segment alone is correct.)
- *=clean-error-message= neutralizes ALL control bytes* (NUL/BEL/…), not just
  whitespace (C++: a provider body can smuggle control chars into the transcript).
- *No-oracle message is now name-free* (=Unknown tool=, no name). Haskell's =Eq=
  on the response type FAILED on the name-bearing message — "identical response"
  is only literally true if name-free; also stops reflecting attacker input.
- *=openai.scm= label =[Provider …]= → =[OpenAI …]=* (was the literal word
  "Provider" while gemini/ollama used their name; C++ + Clojure flagged).

Clarified in spec (every typed port surfaced independently):
- *JSON null vs empty-object vs absent are THREE states* — drives MCP =id: null=
  (reply) vs absent =id= (notification) and =capabilities = {}=. (6/6 ports.)
- *JSON int coercion* contract (=20= / =20.0= / ="20"=; reference rounds, typed
  ports may reject non-integral; MUST NOT crash).
- *Error excerpt* = ≤200 *bytes*, UTF-8-safe, control-neutralized.
- *Retry set is exactly 408/429/5xx*; unlisted 4xx → permanent/non-retried.
- *No-oracle MUST NOT include the tool name*; trust lattice is a *chain*
  (join/meet = max/min); CDATA =unwrap= must parse sections (the breakout marker
  itself contains =]]>=). Fixed the inverted §12/§13 section order.

Regression: =tests/test-hardening.scm= (23, incl per-token safe-path + control
bytes); =scripts/mcp-smoke.sh= (name-free no-oracle).

* What the ports did NOT change
The agent-loop surface (repl, sessions/compaction, streaming, hooks, telemetry,
mcp-client) was stubbed in every port — confirming §12: the *boundaries* are the
hard contract; that surface is where each impl legitimately differs.
