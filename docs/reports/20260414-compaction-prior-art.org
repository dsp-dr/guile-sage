#+TITLE: Prior Art Missing from the Compaction Survey: A Counter-Report
#+AUTHOR: Prof. Marcus "Spark" Wellington
#+DATE: 2026-04-14
#+PROPERTY: header-args :eval never

* Preamble

The companion survey (=20260414-compaction-survey.org=) does commendable
work cataloguing eight contemporary systems and four recent papers.
It does not, however, cite a single result published before 2023.

This is not a minor omission. Every strategy described in sage's
compaction module -- truncation, budget-based eviction, importance
scoring, summarization, intent extraction -- was studied under a
different name decades before the term "context window" entered the
vernacular. This counter-report traces those intellectual lineages.
The goal is not pedantry; it is to connect the compaction design space
to mature theory so that future work can draw on proven results instead
of rediscovering them from first principles.

* 1. Garbage Collection Theory (McCarthy 1960)

John McCarthy's original Lisp garbage collector (CACM, April 1960)
solved the same fundamental problem: a program generates more
objects than memory can hold; which objects should be reclaimed?
Context compaction IS garbage collection for conversational state.

The parallels are structural, not metaphorical:

| GC Concept                   | Compaction Analogue                        |
|------------------------------+--------------------------------------------|
| Root set                     | System prompt + most recent user turn       |
| Reachable objects            | Messages referenced by the active intent    |
| Unreachable objects          | Stale assistant turns with no forward refs  |
| Mark-and-sweep               | Importance scoring (mark), then eviction    |
| Copying collector            | Summarization (copy live data to new space) |
| Generational hypothesis      | Recency bias in ~compact-importance~        |

** Generational GC (Ungar 1984, Appel 1989)

The generational hypothesis -- "most objects die young" -- maps
directly to conversation dynamics. Most assistant responses are
ephemeral (acknowledged and consumed). The few that persist are
architectural decisions, error resolutions, and user preferences.

sage's ~compact-importance~ applies a recency bonus, which is an
ad hoc generational policy. Generational GC theory provides a
principled framework: partition messages into generations, collect
the young generation frequently and cheaply (truncation), and the
old generation rarely and expensively (summarization). This is
exactly what ~compact-auto~ does with its ratio-based strategy
selection, but without citing or formalizing the connection.

*Recommendation*: Frame sage's auto-selection as a two-generation
collector. Young generation = messages newer than the compaction
threshold, collected via truncation. Old generation = messages that
survived one compaction, collected via summarization. This makes the
design legible to anyone who has read Appel's "Simple Generational
Garbage Collection and Fast Allocation" (1989).

* 2. Virtual Memory and Paging (Fotheringham 1961, Denning 1968)

The survey cites MemGPT and correctly notes its virtual memory
metaphor. It does not trace the metaphor to its source.

Fotheringham's Atlas computer (1961) introduced demand paging:
the illusion of a memory space larger than physical RAM, with
transparent page-in/page-out managed by the OS. MemGPT's
core/recall/archival tiers are a literal reimplementation of
this three-level storage hierarchy (registers/RAM/disk).

Peter Denning's working set model (1968) is directly applicable:
the working set of a process at time $t$ with window $\tau$ is
the set of pages referenced in the interval $[t - \tau, t]$.
Pages outside the working set are eviction candidates. sage's
sliding window truncation (~compact-truncate~ with ~#:keep N~)
implements a fixed-window working set policy.

** Page Replacement Algorithms

The survey mentions no page replacement policy by name. Yet:

| Policy       | Year | Compaction Equivalent                        |
|--------------+------+----------------------------------------------|
| FIFO         | 1961 | ~compact-truncate~ (drop oldest)             |
| LRU          | 1966 | Recency-weighted ~compact-importance~        |
| LFU          | 1970 | Keyword-frequency scoring in importance      |
| Clock/Second | 1969 | Could inform a "referenced" bit per message  |
| ARC          | 2003 | Adaptive balance between recency and frequency|
| LIRS         | 2002 | Low inter-reference recency set; accounts    |
|              |      | for access pattern, not just recency         |

Megiddo and Modha's ARC (Adaptive Replacement Cache, 2003) is
particularly relevant: it dynamically balances LRU and LFU by
maintaining ghost entries for recently evicted pages. Applying
this to compaction: after evicting a message, keep a ghost
(topic tag + timestamp). If the user references that topic again,
shift the balance toward keeping topically relevant messages
longer. This would make sage's strategy selection adaptive without
requiring the full trajectory analysis of ACON.

* 3. Lossy Compression and Rate-Distortion Theory (Shannon 1948)

Shannon's rate-distortion function R(D) gives the minimum bit rate
at which a source can be encoded such that distortion does not exceed
D. Context compaction is lossy compression of conversational state.
The survey discusses compression ratios and "comprehension debt" but
never frames this as an information-theoretic problem.

The key insight from rate-distortion theory: there is a provable
lower bound on how much you can compress without exceeding a given
distortion level. For context compaction, this means:

1. *Compression ratio alone is not a quality metric.* A 90%
   reduction with 5% distortion may be strictly superior to an 80%
   reduction with 15% distortion. sage's ~evaluate-compaction~
   weights compression ratio at 20% of the overall score, which
   rewards aggressive compression without bounding distortion.

2. *The distortion measure matters.* sage uses keyword overlap
   (Jaccard-like) and intent string match. These are crude proxies.
   Rate-distortion theory says you must define your distortion
   function /first/, then derive the optimal encoder. The Factory.ai
   probe-based evaluation is closer to a principled distortion
   measure because it tests functional capability, not surface
   similarity.

3. *Source coding theorem applies.* You cannot losslessly compress
   below the entropy of the source. For conversation, this means
   highly entropic exchanges (diverse topics, many facts) have a
   floor on achievable compression. sage should estimate source
   entropy (topic diversity, information density) before selecting
   a compression ratio target.

*Recommendation*: sage's ~evaluate-compaction~ should separate the
compression ratio from the quality score entirely. Report them as
two independent axes -- rate and distortion -- rather than combining
them into a single number. This prevents the optimizer from gaming
the score via aggressive compression.

* 4. Automatic Summarization (Luhn 1958)

Hans Peter Luhn published "The Automatic Creation of Literature
Abstracts" in the IBM Journal of Research and Development in 1958.
His method: score sentences by the frequency and proximity of
"significant" words (words that are neither too common nor too
rare), then extract the highest-scoring sentences.

sage's ~extract-topics~ function counts word frequency and selects
the top N. This is literally Luhn's algorithm, sixty-eight years
later, without citation.

The field has not been idle since 1958:

| Method                  | Year | Contribution                             |
|-------------------------+------+------------------------------------------|
| Luhn (frequency)        | 1958 | Sentence extraction by word significance |
| Edmundson (cue phrases) | 1969 | Added position, cue words, title words   |
| LSA (Gong & Liu)        | 2001 | Latent semantic analysis for salience    |
| LexRank (Erkan)         | 2004 | Graph-based centrality for extraction    |
| TextRank (Mihalcea)     | 2004 | Unsupervised keyword extraction via PR   |
| BertSum (Liu & Lapata)  | 2019 | Transformer-based extractive summary     |

sage's ~compact-summarize~ does not actually summarize -- it
produces a template with metadata (message count, topic list).
This is closer to Edmundson's cue-phrase approach than to any
modern abstractive method. If sage ever adds LLM-based
summarization (calling the model to produce a summary), it enters
the domain of abstractive summarization, where hallucination risk
is well-studied (Maynez et al., "On Faithfulness and Factuality
in Abstractive Summarization," ACL 2020).

* 5. Frames and Salience (Minsky 1974)

Marvin Minsky's frame theory ("A Framework for Representing
Knowledge," MIT AI Memo 306, 1974) proposed that knowledge is
organized in stereotyped structures -- frames -- with slots that
carry default values and can be overridden.

sage's ~score-message-importance~ is frame-slot salience scoring.
The keywords list (="decided"=, ="error"=, ="solution"=, etc.) are
cue words for recognizing which frame a message belongs to. The role-
based base score (system=90, user=50, assistant=40) assigns salience
by frame type. The tool-call bonus recognizes the "action frame."

Minsky's contribution was not just the data structure but the idea
that /default reasoning/ saves computation: you do not re-derive what
is typical; you only store and process deviations. Applied to
compaction: instead of scoring every message, maintain a "default
conversation frame" and only preserve messages that deviate from the
expected flow. A routine Q&A exchange can be compressed to "5 routine
exchanges about X" while a surprising result or error must be kept
verbatim.

* 6. Scripts and MOPs (Schank 1977, Schank & Abelson 1977)

Roger Schank's script theory (Schank & Abelson, "Scripts, Plans,
Goals, and Understanding," 1977) proposed that humans understand
narrative by matching it to stereotyped event sequences -- scripts.

sage's ~compact-intent~ strategy extracts the "intent" from a
conversation. This is script recognition. Schank's later work on
Memory Organization Packets (MOPs, 1982) generalized scripts into
hierarchical, composable structures. The key insight: understanding
requires knowing which /script/ is active, not remembering every
line of dialogue.

sage's ~identify-intent~ function searches for keywords
("implement," "fix," "test," "debug," "explain," "analyze") to
classify the conversation's purpose. This is a flat keyword-based
script recognizer. Schank's framework offers a richer alternative:
scripts have scenes, scenes have actions, actions have preconditions
and postconditions. A "debug" script, for example, has an
"observe failure" scene, a "form hypothesis" scene, a "test
hypothesis" scene, and a "confirm fix" scene. Compaction could
retain the current scene and discard completed scenes, preserving
narrative structure rather than just keywords.

*Recommendation*: Refactor ~compact-intent~ to model conversation
phases (Schank's "scenes") rather than a flat intent label. This
would produce more structured summaries and better support multi-step
tool chains where the intent evolves.

* 7. Working Memory (Baddeley & Hitch 1974, Miller 1956)

George Miller's "The Magical Number Seven, Plus or Minus Two"
(Psychological Review, 1956) established that human working memory
holds approximately 7 chunks. Baddeley and Hitch (1974) refined
this into a multi-component model:

| Component              | Role                           | Compaction Analogue          |
|------------------------+--------------------------------+------------------------------|
| Central executive      | Attention control, switching   | ~compact-auto~ strategy sel. |
| Phonological loop      | Verbal rehearsal (recency)     | Sliding window of recent msg |
| Visuospatial sketchpad | Spatial/visual working memory  | Tool output / code context   |
| Episodic buffer        | Integration across modalities  | System prompt + intent       |

The context window IS the LLM's working memory. The 7 +/- 2 limit,
when applied to "chunks" rather than items, suggests that compaction
should aim for approximately 7 semantically coherent chunks rather
than a fixed token count. sage's ~compact-truncate~ keeps a fixed
number of /messages/, which conflates items with chunks.

Ericsson and Kintsch's "Long-Term Working Memory" theory (1995)
is also relevant: experts circumvent the 7-item limit by encoding
retrieval cues in working memory that point to elaborated structures
in long-term memory. This is precisely what MemGPT's archival memory
search does, and what sage lacks: the ability to hold a retrieval
cue (topic tag, session ID) in context that can be used to recall
detailed information on demand.

* 8. Cache Replacement Policies (Belady 1966, Denning 1968)

Belady's optimal algorithm (1966) evicts the page that will not be
used for the longest time in the future. It is unimplementable in
general (requires future knowledge) but serves as the theoretical
upper bound for all replacement policies.

For conversation compaction, a Belady-like oracle would evict the
message least likely to be referenced in future turns. While we
cannot predict future references exactly, we can approximate:

- Messages about completed sub-tasks are unlikely to be re-referenced
- Error messages for bugs that have been fixed are safe to evict
- Architectural decisions may be re-referenced at any future point
- User preferences are always potentially relevant

sage's importance scoring partially captures this (errors score
high, which is wrong after the error is resolved; system messages
score highest, which is right). A time-varying importance score --
where a message's importance decays as its topic is resolved --
would better approximate Belady's optimum.

The survey's omission of Belady is particularly glaring because his
result proves that /no online algorithm can be optimal/. This means
sage's ~compact-auto~ will always make suboptimal eviction decisions
for some conversation patterns. The proper response is not to seek
a perfect heuristic but to bound the regret -- how much worse the
heuristic performs compared to the optimal offline policy.
Sleator and Tarjan (1985) showed that LRU's competitive ratio is
$k$ (where $k$ is the cache size), meaning it is at most $k$ times
worse than optimal. Similar analysis should be applied to compaction
strategies.

* 9. Cognitive Load Theory (Sweller 1988)

John Sweller's cognitive load theory distinguishes:

- *Intrinsic load*: complexity inherent to the material
- *Extraneous load*: complexity from poor presentation
- *Germane load*: complexity that contributes to learning

Applied to LLM context, intrinsic load is the conversation's actual
information content. Extraneous load is verbose tool outputs, ANSI
escape codes, redundant greetings, and formatting noise. Germane
load is the reasoning chain that the model needs to maintain
coherence.

sage's compaction strategies do not distinguish these load types.
The ~compact-truncate~ strategy removes oldest messages regardless
of load type. A load-aware strategy would:

1. Remove extraneous load first (strip formatting, condense verbose
   tool outputs -- the JetBrains observation masking finding)
2. Preserve germane load (reasoning chains, decision rationale)
3. Reduce intrinsic load last (summarize actual content only when
   the other two sources are exhausted)

This provides a principled ordering that the survey's recommendation
for "observation masking" (R2) partially captures but does not
ground in theory.

* 10. Relevance Theory (Sperber & Wilson 1986)

Dan Sperber and Deirdre Wilson's relevance theory from pragmatics
holds that human communication is governed by the principle of
relevance: every utterance carries an implicit guarantee that it
is worth the processing effort. The "optimal relevance" of a
message is the greatest cognitive effect for the least processing
effort.

This maps to compaction directly: a message should survive
compaction if and only if its expected contribution to future
turns (cognitive effect) exceeds the cost of retaining it
(processing effort, i.e., tokens consumed). sage's importance
score is a rough proxy for cognitive effect but does not account
for processing cost (long messages cost more tokens to retain but
may not proportionally increase their contribution).

*Recommendation*: Weight importance by token efficiency:
~adjusted_score = importance / token_count~. This naturally
penalizes verbose messages that carry little information, which
is exactly what Sperber and Wilson's theory predicts should
happen in rational communication.

* Concluding Remarks

The compaction survey is a useful engineering document. It answers
the question "what are other LLM agent frameworks doing in 2025-2026?"
competently. But it treats context compaction as if it were invented
in 2023 alongside MemGPT.

It was not. The problems of bounded memory, lossy compression,
importance-based retention, automatic summarization, and adaptive
eviction have been studied -- with formal results, proven bounds,
and decades of empirical validation -- since the 1950s. The LLM
context window is not a novel abstraction; it is a fixed-size cache
with a natural language interface.

The practical consequence of this amnesia is that sage's compaction
module reinvents solutions without the benefit of the theory that
produced them. Generational GC tells you /how/ to partition messages.
Rate-distortion theory tells you /when/ to stop compressing.
Belady's theorem tells you that /no online strategy is optimal/ and
gives you the benchmark to measure against. Working memory theory
tells you to think in /chunks, not tokens/. Relevance theory tells
you to weight importance /by cost/.

None of these insights require new code. They require reading papers
that were written before most of us were born.

Please cite your predecessors.
