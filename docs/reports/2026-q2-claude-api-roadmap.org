How to add Anthropic Claude-API features to guile-sage *incrementally*,
choosing per-feature between the *direct Anthropic wire* (the Guile
=claude-api= skill:
[[https://github.com/aygp-dr/anthropic-skills-claude-api-guile][anthropic-skills-claude-api-guile]])
and the *LiteLLM generic wrapper* (OpenAI-shape, already wired). The
LiteLLM column is the user's leak analysis — where the generic wrapper
does *not* fully carry a feature.

Legend (from the leak table):
=N= native/clean · =S= small mapping · =L= leaks (partial / provider-uneven)
· =H= harness-owned (LiteLLM never does it — guile-sage must).

L2 is why we keep LiteLLM in the path for *governed* runs: it gives
virtual keys + spend + guardrails for free. For *isolated* tests we go
direct (avoid the dot-product).

* 3. L3 — agent runtime (HARNESS-OWNED — guile-sage generalizes managed-agents)

LiteLLM passes schemas, never runs the loop. These are guile-sage's to
own — and they map onto the Anthropic *Managed Agents* surface, which we
mirror *locally* (Bastille jails + the aq coordinator instead of
Anthropic-hosted containers).

| Feature (managed-agents surface) | guile-sage analog |
|----------------------------------+-------------------|
| agent loop / control flow        | the REPL multi-step tool chain (governance tuple drives it) |
| tool registry + execution        | =(sage tools)= registry + execute-tool (we run tools; LiteLLM never does) |
| memory                           | ROADMAP — scratch store is the seed; embeddings + recall |
| environments / sandboxes         | RFC-003 — Bastille jails / sandbox-exec / sbx / Deno (our self-hosted) |
| multiagent coordination          | aq / the Issuer / petclinic coordinator |
| events / webhooks                | ROADMAP — hooks.scm is the local event spine |
| scheduled deployments            | cron routines (the /loop + schedule skill) |
| outcomes / eval                  | eval-sage.sh + skill-eval lineage |

* 4. Incremental order (what to land first)

1. *token counting* (L0, native) — accuracy now; cheap; unblocks budget.
2. *prompt caching* (L1) — biggest cost win; stable system+tools prefix.
3. *anthropic-direct provider* (=SAGE_PROVIDER=anthropic=) — the Guile
   wrapper's =/v1/messages= path, so we can A/B direct vs LiteLLM.
4. *structured output* (=output_config.format=) — hardens eval cases.
5. *batches* — run the eval suite at 50% offline.
6. *memory* (L3) — embeddings + scratch recall (the managed-agents memory
   analog, harness-owned).

Each is a small, testable slice. Direct-Anthropic features come from the
Guile skill repo; governed runs stay on LiteLLM; L3 is ours regardless of
provider.
