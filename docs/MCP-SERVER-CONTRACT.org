* 1. Observed wire protocol (empirical, 2026-06-25)

Captured by driving the reference =@modelcontextprotocol/server-everything stdio=
with hand-fed JSON-RPC. This is the protocol we must speak.

- *Framing:* newline-delimited *JSON-RPC 2.0* — one JSON object per line on
  stdin/stdout. NOT LSP =Content-Length= framing.
- *Transports:* server-everything offers =stdio | sse | streamableHttp=. *v1
  scope = stdio only* (the two dogfood clients spawn a stdio subprocess).
- *Handshake (observed):*
  1. client → =initialize= ={protocolVersion, capabilities, clientInfo}=
  2. server → =result{protocolVersion:"2025-06-18", capabilities:{tools:{listChanged}}, serverInfo:{name,title,version}, instructions:"…"}=
  3. client → =notifications/initialized= (no id, no response)
  4. client → =tools/list= → server → =result.tools[]= each
     ={name, title, description, inputSchema(JSON-Schema draft-07), execution?}=
  5. client → =tools/call {name, arguments}= → server → =result.content[]= (e.g.
     ={type:"text", text:…}=)
- The server may emit unsolicited notifications (=notifications/tools/list_changed=);
  optional for us in v1.
- =result.instructions= is passed to the *calling* LLM — a place to state
  guile-sage's safety/taint posture to the caller.

* 2. v1 message contract (what sage MUST implement)

| Method                       | Sage response (v1) |
|------------------------------+--------------------|
| =initialize=                 | =protocolVersion= (echo client's if supported, else ours); =capabilities:{tools:{}}=; =serverInfo:{name:"guile-sage", version:(version-string)}=; =instructions= = sage taint/safety note |
| =notifications/initialized=  | none (notification)                |
| =tools/list=                 | map =*tools*= → ={name, description, inputSchema}= (=parameters= already JSON-schema); only the EXPOSED set (see B2) |
| =tools/call {name,arguments}= | dispatch to the tool's exec-fn; return ={content:[{type:"text",text:<result>}]}=; on denied/unknown/throw → JSON-RPC =error= |
| anything else                | JSON-RPC error =-32601 method not found= |

* 3. Boundary — sage as callee (inverse trust)

When sage is the *server*, the caller is semi-trusted and the host privileges are
*ours*. The boundary inverts vs the client case:

- *B1 — stdout is sacred.* stdout carries ONLY JSON-RPC. Every banner, log,
  warning, =format #t= must go to *stderr* or a file. A stray byte on stdout
  corrupts the stream. (mcp-server mode must suppress the REPL banner + route
  logging off stdout.)
- *B2 — default-safe tool exposure.* By default expose ONLY =*safe-tools*=
  (read-only). Mutating tools (=write_file=, =git_commit=, =eval_scheme=, …) are
  withheld unless explicitly opted in (e.g. =SAGE_MCP_EXPOSE_UNSAFE=1=). An
  external caller must not get sage's write/exec privileges just by asking — this
  is the inverse of the client-side "MCP tools require YOLO" rule.
- *B3 — caller arguments are untrusted input.* Validate =arguments= against the
  tool's =inputSchema=; apply the same path-containment / taint rules as any
  ingress. Never =eval= caller-supplied strings.
- *B4 — results are data, returned verbatim-in-CDATA-equivalent.* Tool output
  goes back as =content[].text=; no secrets/host paths leaked beyond what the
  tool already returns. (Provenance/taint applies to what the tool *reads*, as
  today.)
- *B5 — no fork.* Pure stdin/stdout read-print loop; no =system*=/=open-pipe*=
  (spawn-segfault boundary, see [[file:CHAOS-TESTING.org][CHAOS-TESTING.org]]). Tools that themselves shell
  out already use the libc-=system= path.
- *B6 — capabilities honesty.* Advertise only what we implement (=tools= in v1;
  no =prompts=/=resources=/=sampling= until built). Negotiate =protocolVersion=.

* 4. Conformance (the wizard target + later the real test)

A conformant sage MCP server, driven by the §1 probe sequence, MUST:
- N1 answer =initialize= with =serverInfo.name="guile-sage"= + =capabilities.tools=.
- N2 answer =tools/list= with only the exposed (default-safe) set, each carrying a
  valid draft-07 =inputSchema=.
- N3 answer =tools/call= for a safe tool with =content[].text=; reject an
  un-exposed/unsafe tool with a JSON-RPC error (B2).
- N4 emit *nothing* on stdout except JSON-RPC (B1) — assert no banner/log leaks.
- N5 reject malformed/over-permission =arguments= (B3).
Probe harness = the newline-JSON sequence used in §1, re-pointed at sage.

* 4b. The echo test (grind it — easiest to remember)

Same shape as =@jwalsh/mcp-server-qrcode='s "NPM Package CLI": pipe one JSON-RPC
line in, =jq= out. No =initialize= needed for a single call.

#+begin_src sh
# list tool names (THE echo test)
echo '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}' \
  | guile3 -L src -c '(use-modules (sage main)) (main (list "sage" "mcp-server"))' \
  | jq -r '.result.tools[].name'

# call a safe tool
echo '{"jsonrpc":"2.0","id":1,"method":"tools/call","params":{"name":"git_status","arguments":{}}}' \
  | guile3 -L src -c '(use-modules (sage main)) (main (list "sage" "mcp-server"))' \
  | jq -r '.result.content[0].text'
#+end_src

Full grind (initialize + list + safe call + unsafe-rejected + bad-name):
=gmake mcp-smoke= (=scripts/mcp-smoke.sh=; set =SERVER="sage mcp-server"= to grind
the installed binary, or =SERVER="mcp-server-qrcode"= to grind any stdio server).

* 5. Run / wire (dogfood: Claude Code + Emacs)

One stdio entrypoint: =sage mcp-server= (installed binary; =gmake install=).
Do NOT wrap it in =gmake -C … mcp-server= — gmake prints "Entering directory" to
stdout and corrupts the JSON-RPC stream (B1). Verified clean as a one-shot, same
shape as =npx @jwalsh/mcp-server-qrcode=:

#+begin_src sh
echo '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}' \
  | sage mcp-server | jq -r '.result.tools[].name'
#+end_src

How a consumer adds it via config:
- *Claude Code (installed):* =claude mcp add sage -- sage mcp-server=, or in
  =.mcp.json=: ={"mcpServers":{"sage":{"command":"sage","args":["mcp-server"]}}}=.
- *Claude Code (in-tree, no install):* the dev form in this repo's =.mcp.json=
  (=guile3 -L src -c '(use-modules (sage main)) (main (list "sage" "mcp-server"))'=).
- *Emacs:* =mcp.el= / =gptel= MCP client pointed at =sage mcp-server=.

Prereq for the installed form: =gmake install= (puts =sage= on =~/.local/bin=;
the wrapper runs =guile --no-auto-compile= so a stale global ccache can't shadow
it — the v1.0.1-banner fix). Default-safe: 24 read-only tools; unsafe behind
=SAGE_MCP_EXPOSE_UNSAFE=1=.

* 6b. Worked reference — =@jwalsh/mcp-server-qrcode= (real third-party server)

A small real server we drove end-to-end over stdio (2026-06-25) as a concrete
template for our own. Install: =npm i -g @jwalsh/mcp-server-qrcode= → bin
=mcp-server-qrcode=.

- =initialize= → =serverInfo {name:"QR Code Generator", version:"0.3.6"}=,
  caps =resources/prompts/tools=.
- =tools/list= → one tool =generate-qrcode=, =inputSchema= (draft-07): =content=
  (string), =errorCorrectionLevel= (enum L/M/Q/H, default M), =size= (number
  1–10, default 3), =format= (enum =image|text=, default image).
- =tools/call= success → =result.content= = *two* =text= blocks ("QR code
  generated for: …" + the ASCII QR). So =content[]= is a LIST and may hold
  multiple blocks / mixed types (=text=, =image= with =mimeType=+base64 =data=).
- =tools/call= with a bad arg (=format:"ascii"=, or wrong key =data=) →
  JSON-RPC *error =-32602= "Input validation error"* with the zod failure
  detail. This is exactly our *B3* (validate =arguments= against =inputSchema=)
  and the error code to emit.

Lessons baked into the contract: emit =content= as a list (allow multiple/mixed
blocks); validate args and return =-32602= on schema failure; advertise only
real capabilities.
