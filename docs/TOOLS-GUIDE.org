* Overview

Sage provides *both* native Scheme tools (first-class functions with JSON
schemas, in-process) *and* an MCP client. Native tools are the default for
in-tree capabilities; MCP integrates external tool servers.

* MCP integration (SUPPORTED)

Sage is an MCP (Model Context Protocol) *client*: it discovers SSE MCP servers
from =~/.claude.json= and calls their tools over JSON-RPC 2.0 (=mcp-init=,
=mcp-call-tool= in =src/sage/mcp.scm=). MCP tools register through the same
=register-tool= path as native tools, so their results traverse the same taint
boundary (=guard-tool-result= → =wrap-tool-result=; see
[[file:CHAOS-TESTING.org][CHAOS-TESTING.org]]). MCP tools default to *unsafe*
(require YOLO mode / confirmation). Full invariants:
[[file:MCP-CONTRACT.org][MCP-CONTRACT.org]].

Native vs MCP is a *choice*, not an exclusion: prefer native for in-process,
zero-dependency tools; use MCP to reach external/third-party tool servers.

* Native Tool Building

** Basic Registration

#+begin_src scheme
(register-tool
 "tool_name"
 "Description"
 '(("type" . "object")
   ("properties" . (("arg1" . (("type" . "string")))))
   ("required" . #("arg1")))
 (lambda (args)
   (format #f "Result: ~a" (assoc-ref args "arg1"))))
#+end_src

** Structured Responses

#+begin_src scheme
(register-tool
 "get_contact"
 "Get contact as JSON"
 '(("type" . "object")
   ("properties" . (("name" . (("type" . "string"))))))
 (lambda (args)
   (json-write-string
    `(("name" . ,(assoc-ref args "name"))
      ("email" . "user@example.com")
      ("type" . "contact")))))
#+end_src

** vCard Generation

#+begin_src scheme
(register-tool
 "create_vcard"
 "Create vCard contact"
 '(("type" . "object")
   ("properties" . (("name" . (("type" . "string")))
                    ("email" . (("type" . "string")))
                    ("phone" . (("type" . "string")))
                    ("org" . (("type" . "string")))))
   ("required" . #("name")))
 (lambda (args)
   (format #f "BEGIN:VCARD
VERSION:3.0
FN:~a
EMAIL:~a
TEL:~a
ORG:~a
END:VCARD"
           (assoc-ref args "name")
           (or (assoc-ref args "email") "")
           (or (assoc-ref args "phone") "")
           (or (assoc-ref args "org") ""))))
#+end_src

** QR Code (No MCP Needed)

#+begin_src scheme
;; IMPORTANT: argv-based (no shell). Never shell-interpolate
;; attacker-controlled strings. See ADR-0003 and bd guile-sage-9j7.
(register-tool
 "qrcode"
 "Generate QR code ASCII art"
 '(("type" . "object")
   ("properties" . (("text" . (("type" . "string"))))))
 (lambda (args)
   (let ((text (assoc-ref args "text")))
     ;; capture-argv-in-dir is defined in src/sage/tools.scm; it
     ;; fork+exec's PROG with ARGS as argv and returns stdout.
     (capture-argv-in-dir (workspace) "qrencode" "-t" "UTF8" text))))
#+end_src

** Dynamic Tool Creation

#+begin_src scheme
;; At runtime via create_tool
(execute-tool "create_tool"
  '(("name" . "greet")
    ("description" . "Say hello")
    ("code" . "(lambda (args) (format #f \"Hello, ~a!\" (assoc-ref args \"name\")))")))
#+end_src

* Tool Patterns

** File Operations
#+begin_src scheme
(if (safe-path? path)
    (call-with-input-file (string-append (workspace) "/" path) get-string-all)
    "Unsafe path")
#+end_src

** Shell Commands (prefer argv, never string-interpolate user data)
#+begin_src scheme
;; DO: use capture-argv-in-dir (from src/sage/tools.scm). It fork+exec's
;; the program with arguments as separate argv elements — no /bin/sh is
;; involved, so shell metachars in arguments are treated literally.
(capture-argv-in-dir (workspace) "cmd" "arg1" "arg2")

;; DON'T: `(system (format #f "cmd ~a > /tmp/out" user-input))`.
;; Any metachar in user-input (;, |, &, `, $, newline) gives the
;; caller arbitrary shell execution. See bd guile-sage-9j7 / 07f.
#+end_src

** HTTP Requests
#+begin_src scheme
(let ((result (http-get url)))
  (if (= (car result) 200) (cdr result) "Error"))
#+end_src

* MCP vs Native (both supported — pick per use case)

| Aspect | MCP (external) | Sage Native (in-tree) |
|--------|----------------|------------------------|
| Latency | SSE + JSON-RPC over network | In-process |
| Dependencies | the external MCP server | Pure Scheme |
| Debugging | =/logs= + server logs | =/logs= |
| Extensibility | add a server to =~/.claude.json= | =register-tool= lambda |
| Safety default | unsafe (YOLO/confirm) | per-tool =safe?= flag |

Use *native* for in-process, dependency-free tools; use *MCP* to reach
external tool servers. See [[file:MCP-CONTRACT.org][MCP-CONTRACT.org]].
