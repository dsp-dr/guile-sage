* Where sage leads

1. *Local-first with Ollama* — the only tool in this matrix that
   natively targets local Ollama without a cloud key. Aider supports
   Ollama via OpenAI-compat but treats it as a secondary path.

2. *Dynamic model-tier selection* — auto-switches between fast and
   standard models based on context size. No other tool does this.

3. *Graceful model removal* — survives =ollama rm= without crashing.
   Picks the smallest chat-capable substitute automatically.

4. *Self-inspection tools* — =read_logs=, =search_logs=, =log_stats=,
   =log_timeline= let the agent introspect its own execution history.
   No other CLI exposes this.

5. *Anti-hallucination guard* — explicit post-tool-error injection
   that tells the model not to fabricate content. Others rely on
   implicit model behaviour or prompt engineering.

6. *Timing protocol* — a reproducible per-model benchmark script with
   locked-in measurements. No other tool ships this.

7. *CPRR-structured git notes* — every commit carries attribution,
   testing, conjectures, validation, and research pointers. Unique.

8. *Property-based testing (PBT)* — 64+ properties across 6 contract
   layers, with a cross-pollination path to hegel-guile. No other AI
   CLI ships PBT for its own internals.

* Roadmap priorities for 0.7.0

Based on the gap analysis, the highest-impact work for 0.7.0:

1. *Multi-step tool chains* (P0, M) — the dispatch loop re-prompt.
   This is the #1 UX gap per docs/MULTISTEP-CHAINS-0.6.x.md: 0/6
   multi-step prompts succeeded. Every competitor handles this.

2. *=/compact= slash command* (P0, S) — wire the existing 5
   compaction strategies (already in =src/sage/compaction.scm=) to a
   user-invocable command. Low effort, high visibility.

3. *Prompt caching via Ollama* (P0, S) — Ollama supports prompt
   prefix caching natively. Reduce prompt_eval_duration for
   multi-turn sessions. Just reuse the message prefix.

4. *Stdio MCP transport* (P1, M) — unlocks local MCP servers like
   filesystem, github, sqlite that Claude Code users already have
   configured in =~/.claude.json=.

5. *Web search tool* (P0, M) — add a =web_search= tool backed by a
   search API (DuckDuckGo Instant Answer, Brave Search, or SearXNG
   on the LAN). Unblocks research-style prompts.

6. *Plan/read-only mode* (P1, S) — =--plan= flag that registers
   ONLY safe-tools and skips YOLO. Useful for code review sessions.

7. *Auto-memory* (P1, M) — persistent memory backed by XDG files,
   same as Claude Code's auto-memory. Store user preferences,
   project context, and cross-session learning.
