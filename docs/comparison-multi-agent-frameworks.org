* Overview

This document compares guile-sage's architecture against three prominent multi-agent frameworks:
- *MetaGPT* (geekan/MetaGPT) - SOP-based role orchestration
- *CAMEL* (camel-ai/camel) - Role-playing conversation framework
- *AutoGen* (microsoft/autogen) - Actor model with type-driven routing

* Architecture Comparison

** Agent Model

| Framework | Model | State Management | Communication |
|-----------+-------+------------------+---------------|
| sage | Single agent, tool-first | Session-based, beads persistence | IRC channels, file queues |
| MetaGPT | Role FSM with states | RoleContext + Memory + Working Memory | Pub/Sub via Environment |
| CAMEL | ChatAgent with step() | AgentMemory (history + vector) | RolePlaying ping-pong |
| AutoGen | RoutedAgent actor model | Stateful agents, caller manages history | Type-routed messages |

*** MetaGPT

#+begin_src python
# Role FSM with think-act-observe pattern
class Role:
    async def run(self):
        await self._observe()    # Read from msg_buffer
        await self._think()      # LLM selects action
        await self._act()        # Execute action

# Message routing via Environment
Environment.publish_message(message):
    for role in members:
        if is_send_to(message, role.addresses):
            role.put_message(message)
#+end_src

*Key patterns:*
- SOP (Standard Operating Procedure) via role definitions
- Three react modes: REACT (LLM-driven), BY_ORDER, PLAN_AND_ACT
- ActionNode for structured output with review/revise cycles
- Team serialization for checkpointing

* Recommendations

** Adopt from MetaGPT

1. *Structured ActionNode*: sage could benefit from structured output schemas
2. *Cost tracking*: Token budget management integrated with session stats
3. *React modes*: Support LLM-driven vs sequential action selection

** Adopt from CAMEL

1. *ResponseTerminator patterns*: Clean termination conditions
2. *Timeout management*: Per-tool timeout configuration
3. *Toolkit organization*: Group tools by domain (file, git, agent)

** Adopt from AutoGen

1. *Intervention handlers*: External hooks for audit layer (aligns with guile-0ai)
2. *Type-driven routing*: Message type → handler mapping
3. *Topic isolation*: Prevent cross-session pollution

** Keep Unique to sage

1. *Self-modification*: No other framework emphasizes this
2. *Beads integration*: External persistent task memory
3. *IRC coordination*: Simple, debuggable multi-instance communication
4. *Guile/Scheme foundation*: Homoiconicity for meta-programming
