The language-agnostic implementation contract. *Spec version is independent of an
impl's own release version*: =guile-sage= ships release =v1.1.0= and is *spec-v3
compliant*; =sage-clojure= (2nd reference) is likewise spec-v3. The spec advances
by *rounds* — each port round's findings fold forward (§14). Architecture +
boundaries: [[file:docs/ARCHITECTURE.org][docs/ARCHITECTURE.org]]; build protocol: [[file:docs/PORT-BUILD-DIRECTIVE.org][PORT-BUILD-DIRECTIVE.org]].

* 1. Identity & scope
A terminal (REPL/TUI) AI coding agent: an LLM drives a tool-use loop over a
workspace, with sessions, context management, guardrails, and observability. CLI
entry =sage= (interactive default; =-p/--print= one-shot; =sage mcp-server=
subcommand). Runtime: guile3/guild3.

*Reference implementations* (two, kept current with this spec): =guile-sage=
(Guile, primary) and =sage-clojure= (Clojure, second — built to feature parity).
The other language ports are ephemeral conformance probes (§13).

* 2. Providers (LLM backends)
- =SAGE_PROVIDER= ∈ {=ollama=, =gemini=, =openai=}. One active per session.
- *ollama* — local/cloud, =SAGE_OLLAMA_HOST=, NDJSON streaming.
- *gemini* — Google API, =GEMINI_API_KEY=, =:generateContent= / SSE.
- *openai* — OpenAI-shape; the gateway path (LiteLLM / CF AI Gateway):
  =SAGE_OPENAI_BASE=, =SAGE_OPENAI_API_KEY= (proxy/virtual key only).
- Per-provider: chat, streaming, tool-calling, token-usage extraction, model list.
- *Error normalization* (all providers): non-200 → a single clean
  =[<Provider> <label>: <msg>]= assistant line.
  - =<label>= is a *provider-normalized human-readable reason* — phrasing is
    provider-specific (e.g. 404 → gemini "model not found", openai "request
    failed (HTTP 404)"; 401 → "authentication failed (invalid/expired key)").
    The *stable contract* is the classification (below) + the shape, NOT
    byte-identical tokens across providers. Canonical buckets: =connection
    failed= (0), =authentication failed …= (401/403), =rate limited= (429),
    =server error= (5xx), provider-specific or =request failed (HTTP <n>)= else.
  - =<msg>= is a *sanitized, bounded excerpt* of the body: ALL control bytes
    (C0 <0x20 and DEL 0x7f, not just =[ \t\r\n]=) neutralized to spaces and
    collapsed, then truncated to ≤200 *bytes* (UTF-8-boundary-safe — never split
    a multibyte sequence). "Never throw the raw body" means no control chars / no
    newlines / no kilobytes; a short clean excerpt IS allowed.
  - *Classification ≠ retry policy.* Codes are classified permanent (401/403/404
    and any other non-transient code) vs transient (0/408/429/5xx) for the label;
    the *retry* set (§8) is exactly 408/429/5xx — code 0 is transient-classified
    but fails fast, and unlisted 4xx (400/422/…) are permanent / non-retried.
- *Model-tier auto-select*: fast/standard tiers (defaults =llama3.2:latest=,
  ceilings 4000/8000); probe installed models, fall back if missing.

* 3. Tool system
- Registry of ~34 tools (=register-tool name description json-schema exec-fn
  #:safe=); 24 are read-only "safe".
- *Permission model*: safe tools always allowed; mutating tools (write_file,
  edit_file, git_commit, git_push, eval_scheme, …) require YOLO mode or
  confirmation. =SAGE_YOLO_MODE=1= permits all.
- Path containment (=safe-path?=): bare/relative paths resolve against the
  workspace (default-allow workspace-relative); =/tmp/= allowed; empty path
  rejected. Reject *NUL bytes* and any =..= (substring). Sensitive-segment block
  is *per token* (neither substring nor plain exact — both are wrong):
  - =.env=: a segment equal to =.env= OR starting with =.env.= (blocks =.env=,
    =.env.local=, =.env.production=; allows =my.env=).
  - =.git= / =.ssh= / =.gnupg=: exact segment (allows =.gitignore=,
    =.gitattributes=).
  The block applies to *ALL* paths *including =/tmp/=* (so =/tmp/.ssh/id_rsa= is
  rejected). Symlink-safe canonicalization. MUST be *total* (never raises).
- *JSON tool-arg coercion*: a tool wanting an integer must coerce an arg that
  arrives as =20=, =20.0=, or ="20"= (LLM/JSON-RPC clients emit all three). The
  reference rounds near-integral values (LLM-robust); typed ports may instead
  *reject* non-integral (=20.5=) rather than truncate — implementations choose,
  but MUST NOT crash. =null=/absent → tool's default.
- Tool families: file read/list/glob/search; file write/edit (unsafe); git
  status/diff/log/fetch (+ commit/push/add-note unsafe); fetch_url; log
  search/stats (self-inspection); agent tasks (beads-backed); generate_image;
  scratch store; whoami; eval_scheme (unsafe, denylist-sandboxed).

* 4. MCP — bidirectional
- *Client* (=mcp.scm=): discover SSE MCP servers from =~/.claude.json=, call
  tools over JSON-RPC (=mcp-init=, =mcp-call-tool=); register them as tools;
  MCP tools default unsafe.
- *Server* (=mcp-server.scm=, new in 1.1.0): =sage mcp-server= serves the tool
  registry over *stdio newline-JSON-RPC 2.0* (=initialize= / =tools/list= /
  =tools/call= / =ping=); safe-only by default, unsafe behind
  =SAGE_MCP_EXPOSE_UNSAFE=1=; stdout is JSON-RPC only (logs→stderr). Error codes:
  =-32700= (parse error, id null — malformed JSON MUST get a reply, not a silent
  drop), =-32600=, =-32601=, =-32602=. Lets Claude Code / Emacs call sage.
  - *No oracle*: =tools/call= on an unknown tool AND on an unexposed (gated)
    tool MUST return the *byte-identical* response (=-32601=, fixed message
    =Unknown tool= that MUST NOT include the tool name) — a peer must not tell a
    gated tool from a nonexistent one, nor be told how to escalate. Echoing the
    caller-supplied name makes responses differ per call (a typed port's =Eq=
    property caught this). The name + =SAGE_MCP_EXPOSE_UNSAFE= hint go to stderr.
  - *JSON null vs empty-object vs absent are THREE distinct states* (the decoder
    MUST preserve the distinction even though the reference's alist collapses
    them): an absent param ≠ explicit =null= ≠ ={}=. This drives wire semantics:
    a request with =id: null= demands a reply, an *absent* =id= is a
    notification (no reply); =initialize.capabilities= is ={}= (empty-object),
    not =null=. (Every typed port surfaced this independently.)
  - *Framing*: blank / whitespace-only stdin lines are transport framing, not
    messages — skipped silently (NOT =-32700=). =-32700= is only for non-blank
    input that fails to parse.
  *Taint scope*: served results are *plain MCP content* — the taint envelope
  (§7) is for sage's OWN-LLM path, NOT the server wire; a peer client owns its
  own trust boundary (a server-side guard may sanitize but must not wrap).

* 5. Agent loop & REPL
- Interactive REPL: prompt → provider chat (streaming or not) → tool-call loop
  (execute → feed results back → repeat to a cap) → final answer.
- Slash commands: /help /version /tools /mcp /model(s) /sessions /save /load
  /reset /stats /stream /debug /logs /reload /tier /doctor /agent /tasks
  /compact /define-command + custom commands.
- =SAGE_STREAMING=0= forces non-streaming.

* 6. Sessions, memory, context
- Sessions persisted as JSON under XDG (=~/.local/share/sage/sessions/=):
  name, model, messages, stats (tokens, request/tool counts). File-based.
- Project-local context: loads =SAGE.md= / =AGENTS.md=.
- *Context management*: token-limit per model (=*token-limits*=); warnings at
  *75% / 90% / 95%*; compaction strategies (truncate / token-limit / importance
  / summarize / intent), auto-compact at threshold; =/compact [n]=.

* 7. Guardrails & safety (the taint boundary)
- *All external bytes are untrusted data, never instructions.* Single choke
  point: every tool result (native AND MCP) → =guard-tool-result= →
  =wrap-tool-result= → =<tool-result safe="false">= over (for fetches)
  =<fetch-result trust="untrusted" sha256=…>= over CDATA, with =]]>= breakout
  escaping (=]]>= → =]]]]><![CDATA[>=). System prompt declares tool-result blocks
  untrusted. NOTE: the breakout marker itself *contains* =]]>=, so a byte-faithful
  =unwrap= MUST parse and concatenate the CDATA sections — a naive string-replace
  is not an inverse for content that already holds the artifact (the round-trip
  property guards this; every port hit it).
- Trust lattice (=provenance.scm=): =untrusted < provider < local < verified= —
  this is a *total order (chain)*, so join/meet collapse to =max=/=min= and
  bottom/top are =untrusted=/=verified=. The lattice is the OWN-LLM path's, NOT
  the MCP server wire (served results are plain; §4).
- Sandbox/confinement awareness (=sandbox.scm=); see =docs/CHAOS-TESTING.org=,
  =rfc/003= for the ephemeral-key/egress design.

* 8. HTTP transport & resilience
- All HTTP via curl with =--connect-timeout 5=; startup model probe fails fast
  (no retry); chat (=http-post-with-timeout=) retries the *transient set*
  =408/429/5xx= with exponential backoff (=SAGE_HTTP_MAX_RETRIES=, default 2).
  Connection failure (code 0) fails fast (no retry — each retry costs a full
  connect-timeout for no gain).
- *Spawn boundary* (*Guile-runtime-specific* — v3 note, flagged by the Chez
  port): in guile3 shell out via libc =(system …)= only — never =system*= /
  =open-pipe*= / =open-input-pipe= (they SIGSEGV in this long-lived,
  multi-threaded runtime). Streaming = =(system "curl … > FIFO &")= +
  =(open-input-file FIFO)=. This is a Guile constraint, NOT a cross-language
  contract — other runtimes use their native process API (e.g. Chez =(system)=,
  =System.cmd=, =popen=).

* 9. Hooks
- Lifecycle hooks (PreToolUse / PostToolUse) — register guards/observers; can
  veto a tool call. Negative contracts: things the agent MUST NOT do
  (=docs/HOOK-NEGATIVE-CONTRACTS.org=).

* 10. Observability (telemetry)
- OTLP/HTTP metrics (fire-and-forget; no-op if endpoint unset). Counters:
  sessions, token usage (model + in/out), tool decisions, active time, cost.
  =OTEL_EXPORTER_OTLP_*=, =SAGE_TELEMETRY_DISABLE=, =SAGE_DEBUG_HTTP= wire logs.

* 11. Build, test, config conventions
- =gmake= (GNU make; BSD make mis-parses it); =guile3=/=guild3= (auto-detected).
- Targets: =build= / =check= / =help= / =run= / =install= / lint-{code,org,docs,
  contracts} / =chaos-net= / =mcp-smoke=.
- Tests: SRFI-64 suite (=gmake check=, ~40 files) + *property-based testing*
  (=test-pbt.scm=, thousands of trials) + contract/doc lints + a tmux PTY chaos
  harness. Custom runners propagate failures (no masked greens).
- Config precedence (v3 reconcile — *dotenv shadows env*, matching the
  reference's intentional behavior; the Chez port flagged the spec had it
  backwards): loaded *dotenv* (=.env=, =~/.config/sage/.env=) for
  =SAGE_<KEY>=/=<KEY>= → shell env =SAGE_<KEY>= → shell env =<KEY>= → default.
  I.e. a value present in =.env= wins over the shell environment (deliberate, if
  unconventional). XDG dirs.

* 12. Reimplementation notes (port priorities)
A port MUST preserve, in priority order: (1) the *taint boundary* (§7) — single
choke point, untrusted wrapping; (2) the *permission model* (§3) — safe/unsafe +
YOLO + per-token path block; (3) *provider error normalization* + *HTTP
resilience* (§2/§8); (4) the *MCP server + client* contracts (§4 — no-oracle,
served-plain, framing); (5) *context/compaction* thresholds (§6). Surface details
(slash-command names, file layout) may differ; the boundaries and contracts may
not. Verify a port against these as executable assertions — ideally
property-based; a sum type for JSON (null/empty-object/absent) and a total
=safe-path?= are where typed ports surface the spec's blur.

* 15. v4 boundary clarifications (round-4 harvest)
Each amends the same-named section above. Every item is backed by multiple of the
11 round-4 ports; full evidence: =docs/reports/20260627-v4-grind-findings.org=.

The grind's purpose was *spec clarity + coverage of cases not in our own
tests/feature list* — NOT to retrofit the reference to every port's choice. So:
where the reference made a *deliberate call*, this spec states the *reference's*
behavior as authoritative and marks port-suggested alternatives as *optional
refinements* (not gaps). Only genuine *bugs* surfaced by the grind were fixed in
the reference (e.g. the §15.3 EXPOSE_UNSAFE/YOLO footgun). Disagreement is a
recorded finding, not a re-litigation trigger.

** 15.1 Value-type identity (amends §4)
A port SHOULD carry value-type identity: =null= / empty-object =\{}= / *absent* /
number-vs-string distinct at every node. Both extremes converged on it — typed
ports via sum types (Haskell/Rust/Scala/OCaml/Zig), untyped string ports via an
explicit ={TYPE PAYLOAD}= tag (AWK/Tcl), hash-reader langs for free
(Racket/Clojure/Erlang/JS) — which is why it's stated as the clear target.
*Reference (decided):* guile-sage's alist collapses =\{}= and =[]= to ='()=; it
preserves the *wire-critical* states explicitly (=null= sentinel, *absent* via
missing key, =\{}= on the wire via =json-empty-object=). The full per-node
distinction is a *port* target, not retrofitted to the reference alist. Partner
note: encoding is deterministic (alist order) — keep it canonical so the §4
no-oracle byte-identity holds across re-encodes.

** 15.2 MCP error-code taxonomy + no-oracle scope (amends §4)
- *Reference (decided) code set:* =-32700= parse error (id null); =-32601= unknown
  method AND (separately) no-oracle unknown/unexposed *tool*; =-32603= tool-exec
  failure. *Optional refinements* a port MAY add (not in the reference): =-32600=
  for valid-JSON-but-not-an-object, =-32602= for invalid params pre-checked before
  tool lookup. Either is conformant.
- *No-oracle scope (contract):* applies to =tools/call= only; the guarantee is
  "the error *object* is byte-identical given the same =id=, with a name-free
  message". An unknown *method* legitimately differs. Absent/null =arguments= →
  tool defaults; an id-less =tools/call= is a notification (no reply).

** 15.3 safe-path total/effectful split + value-gated flags (amends §3)
- Split the two conflicting requirements: a *pure, total* lexical guard (the
  contract — NUL/=..=/empty/per-token block, never raises) and an *optional,
  best-effort effectful* symlink-canonicalization pass (may do I/O, may fail).
  Ordering is load-bearing: reject =..= (substring; intentional fail-closed on
  benign =my..notes=) BEFORE any canonicalization.
- *Value-gated dangerous flags:* =SAGE_MCP_EXPOSE_UNSAFE= and =SAGE_YOLO_MODE=
  enable only on an affirmative VALUE (=1/true/yes/on=), never mere presence —
  =…=0= must NOT enable. (Fixed in guile-sage; was a footgun.)

** 15.4 Compaction: pure vs LLM-backed (amends §6)
Two classes: *pure/portable* (=truncate=, =token-limit= — deterministic,
property-testable in a boundary port) and *LLM-backed* (=summarize=, =intent=,
=importance= — require a model, NOT unit-testable at the boundary; a port stubs
them honestly). Auto-compact threshold *(reference, decided)* is *0.8 × the model
token limit* (was unspecified before v4 — ports picked 0.8–0.9; the reference's
0.8 is authoritative); warnings stay at 75/90/95%.

** 15.5 HTTP backoff + spawn (amends §8)
- Backoff: exponential, *impl-defined base · 2^n* up to =SAGE_HTTP_MAX_RETRIES=
  (default 2); jitter optional. *Reference (decided):* base 500ms (ports used
  200–500ms — the base is a tuning knob, not a contract). Retry set stays exactly
  408/429/5xx; code 0 fails fast.
- *Spawn:* the rule is generic — *use the runtime's native fork-safe process API*.
  Guile's =system*=/=open-pipe*= SIGSEGV is the *example/motivation*, not a
  cross-language constraint (Babashka/Elisp/Racket/Rust/… confirmed no such class).

** 15.6 Error excerpt: encode-first + label buckets (amends §2)
- "≤200 bytes, UTF-8-safe" means *truncate already-valid UTF-8 at a char boundary*;
  sanitize MUST NOT *introduce* invalidity, and decoded-string runtimes (Tcl/Io/
  Elisp) must *encode to bytes first* before measuring. Empty sanitized body:
  the *reference (decided)* renders =[<Provider> <label>: ]=; omitting the
  trailing =: = is an *optional* cosmetic refinement.
- =<label>= example strings are *illustrative*; the stable contract is the
  *classification bucket* (permanent vs transient), not byte-identical tokens.

** 15.7 Correct CDATA taint invariant + scope cross-ref (amends §7)
- The sound verification is *round-trip identity* + ="]]>"-count == content-count+1=,
  NOT "balanced/nested =<![CDATA[=" (CDATA does not nest; a literal marker in
  attacker bytes is inert). Name a =cdata-no-escape= property alongside round-trip.
- *Cross-ref (resolves an apparent §4↔§7 contradiction):* §7's "EVERY tool result
  is wrapped" is the *own-LLM* path; an MCP *server* returns *plain* protocol
  content (§4 served-plain). The envelope never goes on the server wire.
