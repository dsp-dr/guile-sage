* Pre-Release Testing Process

This document describes the testing process for guile-sage releases.

** Quick Smoke Test

Run the full test suite:
#+begin_src bash
make check
#+end_src

Expected: All tests pass (some security tests may show expected failures
for safe tools like write_file, edit_file, git_commit).

** Manual REPL Testing

*** 1. Start Session
#+begin_src bash
cd /path/to/guile-sage
make repl
#+end_src

Verify:
- [ ] Welcome banner shows correct version
- [ ] Model and host are displayed
- [ ] Prompt shows user@host:repo format

*** 2. Basic Commands
#+begin_src
/help      - Verify all commands listed
/version   - Verify version matches release
/status    - Verify session stats display
/workspace - Verify workspace path
/tools     - Verify all tools listed (should include read_logs, search_logs)
#+end_src

*** 3. Logging (v0.3.0+)
#+begin_src
/logs      - Verify recent log entries displayed
/logs 10   - Verify line limit works
#+end_src

Also verify .logs/sage.log exists and contains entries.

*** 4. Tool Execution
Test that tools work correctly:
#+begin_src
What files are in the src directory?
#+end_src

Sage should use list_files or glob_files tool and return results.

*** 5. Git Tools
#+begin_src
What's the git status?
Show me the recent commits.
#+end_src

Verify git_status and git_log tools execute and return results.

*** 6. Session Management
#+begin_src
/save test-session
/sessions
/load test-session
#+end_src

*** 7. Self-Inspection (v0.3.0+)
Ask sage to debug itself:
#+begin_src
Can you check the logs to see what tools were called recently?
#+end_src

Sage should use read_logs tool and show recent activity.

*** 8. Error Recovery
Trigger an intentional error and verify logging:
#+begin_src
/reload
/logs 20 error
#+end_src

*** 9. Exit
#+begin_src
/exit
#+end_src

** Automated Test Script

For CI/CD, use the following test script:

#+begin_src bash
#!/bin/bash
# tests/smoke-test.sh

set -e

echo "=== Sage Smoke Test ==="

# 1. Run unit tests
make check

# 2. Verify logging module loads
guile3 -L src -c "(use-modules (sage logging)) (init-logging) (log-info \"test\" \"smoke test\")"

# 3. Verify log file created
if [ -f .logs/sage.log ]; then
    echo "PASS: Log file created"
else
    echo "FAIL: Log file not created"
    exit 1
fi

# 4. Verify log content
if grep -q "smoke test" .logs/sage.log; then
    echo "PASS: Log entry written"
else
    echo "FAIL: Log entry not found"
    exit 1
fi

# 5. Clean up
rm -f .logs/sage.log

echo "=== All Smoke Tests Passed ==="
#+end_src

** Version-Specific Tests

*** v0.3.0 - Logging Infrastructure

New features to test:
1. [ ] src/sage/logging.scm module loads
2. [ ] init-logging creates .logs/ directory
3. [ ] log-info/log-warn/log-error write to file
4. [ ] /logs command shows recent entries
5. [ ] read_logs tool works via LLM
6. [ ] search_logs tool works via LLM
7. [ ] Tool calls are logged with duration
8. [ ] API requests/responses are logged
9. [ ] Malformed JSON tool calls are logged as warnings
10. [ ] Session create/load/save events are logged

** Regression Tests

Always verify:
- [ ] Existing unit tests pass
- [ ] REPL starts without errors
- [ ] Basic chat works
- [ ] Tool calling works
- [ ] Session persistence works
- [ ] /reload hot-reload works

** Known Issues

- Security tests for write_file, edit_file, git_commit show as "failed"
  because these are intentionally in *safe-tools* for the dev workflow.
  This is expected behavior, not a bug.

** Release Checklist

1. [ ] Update version in src/sage/version.scm
2. [ ] Run `make check`
3. [ ] Run manual REPL testing
4. [ ] Update CHANGELOG (if exists)
5. [ ] Commit with `chore: Release vX.Y.Z`
6. [ ] Tag release: `git tag -a vX.Y.Z -m "Release vX.Y.Z"`
7. [ ] Push: `git push && git push --tags`
