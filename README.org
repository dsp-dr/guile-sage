*AI REPL with tool calling for Guile Scheme*

** Overview

guile-sage is an interactive AI assistant REPL written in Guile3 Scheme. It provides:

- Chat interface with Ollama-compatible LLMs
- Tool calling system with safety permissions
- Session management with persistence
- Self-modification capabilities
- Context compaction for long conversations

** Features

*** Chat Interface
- Interactive REPL with readline support
- Slash commands for configuration
- Debug mode for development

*** Tool System

Tools are classified as /safe/ (always allowed) or /unsafe/ (require YOLO mode).
The default configuration enables most development tools as safe for interactive use.

**** File Operations
| Tool | Description | Safe |
|------+-------------+------|
| =read_file= | Read file contents | Yes |
| =list_files= | List directory contents | Yes |
| =write_file= | Write content to file | Yes |
| =edit_file= | Search/replace in file | Yes |
| =glob_files= | Find files by pattern | Yes |
| =search_files= | Grep for patterns | Yes |

**** Git Operations
| Tool | Description | Safe |
|------+-------------+------|
| =git_status= | Show git status | Yes |
| =git_diff= | Show git diff | Yes |
| =git_log= | Show git history | Yes |
| =git_commit= | Stage files and commit | Yes |
| =git_add_note= | Add git notes | Yes |
| =git_push= | Push to remote | Yes |

**** Self-Inspection (Logging)
| Tool | Description | Safe |
|------+-------------+------|
| =read_logs= | Read recent log entries | Yes |
| =search_logs= | Search logs for patterns | Yes |

**** Agent Tasks
| Tool | Description | Safe |
|------+-------------+------|
| =sage_task_create= | Create agent task | Yes |
| =sage_task_complete= | Mark task complete | Yes |
| =sage_task_list= | List pending tasks | Yes |
| =sage_task_status= | Get agent status | Yes |

**** Meta/Unsafe (require YOLO mode)
| Tool | Description | Safe |
|------+-------------+------|
| =run_tests= | Execute test suite | No |
| =eval_scheme= | Evaluate Scheme code | No |
| =reload_module= | Hot-reload modules | No |
| =create_tool= | Register new tools | No |

Enable YOLO mode for unsafe tools: =SAGE_YOLO_MODE=1 make run= or =make run-yolo=

*** Session Management
- Conversation history with token tracking
- Save/load sessions to JSON
- Context compaction for long conversations
- Statistics tracking (tokens, requests, tool calls)

** Requirements

- Guile 3.0+
- curl (for HTTP requests)
- Ollama (local or cloud API)

** Installation

#+begin_src bash
# Clone the repository
git clone https://github.com/dsp-dr/guile-sage.git
cd guile-sage

# Initialize configuration
make init

# Verify setup
make check-config
#+end_src

** Configuration

Create a =.env= file based on the template:

#+begin_src bash
cp .env.template .env
#+end_src

Configure your Ollama endpoint:

#+begin_src bash
# Local Ollama
SAGE_OLLAMA_HOST=http://localhost:11434
SAGE_MODEL=qwen3-coder:latest

# Cloud Ollama (with authentication)
SAGE_OLLAMA_HOST=https://api.example.com
SAGE_OLLAMA_API_KEY=your-api-key
#+end_src

*** XDG Directory Structure

guile-sage follows the XDG Base Directory specification:

#+begin_example
$XDG_CONFIG_HOME/sage/     # ~/.config/sage
└── (config files)

$XDG_DATA_HOME/sage/       # ~/.local/share/sage
├── sessions/              # Saved conversation sessions
├── agents/                # Custom agent definitions
├── projects/              # Project-specific configurations
└── commands/              # Custom slash commands

$XDG_CACHE_HOME/sage/      # ~/.cache/sage
└── (cache files)

$XDG_STATE_HOME/sage/      # ~/.local/state/sage
└── (state files)
#+end_example

Override with environment variables:

#+begin_src bash
SAGE_CONFIG_DIR=/custom/config/path
SAGE_DATA_DIR=/custom/data/path
SAGE_SESSION_DIR=/custom/sessions/path
#+end_src

*** Project-Local Sessions

Like Claude Code, guile-sage stores sessions per-project using path encoding:

#+begin_example
/home/user/myproject  ->  -home-user-myproject
#+end_example

Sessions are stored in:
#+begin_example
$XDG_DATA_HOME/sage/projects/-home-user-myproject/sessions/
#+end_example

This keeps conversation history separate per working directory.

** Architecture

*** Beads Methodology

This project follows the beads methodology for development:
- Small, atomic commits
- Progressive commit protocol (PCP)
- Git notes for documentation
- Explicit file staging (never =git add -A=)

*** Self-Modification

guile-sage can modify itself through its tool system:
- =eval_scheme= evaluates arbitrary Scheme code
- =create_tool= registers new tools dynamically
- =reload_module= hot-reloads modified modules
- =edit_file= modifies source files

** Platform Notes

*** FreeBSD

Developed and tested on FreeBSD 14.3. Known workarounds:
- Uses =curl= with temp files instead of =open-input-pipe= (segfault issue)
- Custom JSON parser (guile-json not available)
